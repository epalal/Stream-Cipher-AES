# Stream Cipher Hardware Accelerator (AES Galois Multiplier)

## Overview
This repository contains the RTL design, functional verification, and high-level software model of a custom Stream Cipher based on the Advanced Encryption Standard (AES) Galois Multiplication function. 

## Cryptographic Algorithm
The hardware module supports both continuous stream encryption and decryption. The keystream is generated by applying the AES Galois Multiplication function ($S$) to a 32-bit Counter Block ($CB$). The mathematical law governing the cipher is defined as:

$C[i] = P[i] \oplus S(CB[i])$

where $CB[i] = (K + i) \bmod 2^{32}$, with $K$ being the initial 32-bit symmetric encryption key.

## Architecture & Security Features
The hardware architecture was specifically optimized to guarantee real-time throughput and resilience against cryptographic vulnerabilities:
* **Single-Cycle Datapath:** The strictly combinational unrolled architecture allows the system to process the entire Galois multiplication and XOR encryption in a single clock cycle, achieving a perfectly synchronous Cycle Per Data (CPD) ratio of 1.
* **Secure Handshaking:** The module implements a robust control logic using the `in_valid` and `out_ready` signals. This active gating mechanism freezes the internal counter during stalling or empty messages, successfully preventing **"keystream reuse"**—a severe vulnerability in stream ciphers.

## FPGA Implementation Results
The RTL design was synthesized and physically implemented targeting the Intel **Cyclone V 5CGXFC9D6F27C7** FPGA. The Static Timing Analysis (STA) extracted from the worst-case Process, Voltage, and Temperature (PVT) corner (Slow 1100mV 0C) demonstrated an extremely lightweight and fast footprint:
* **Area:** 64 ALMs
* **Maximum Frequency ($F_{max}$):** 147.54 MHz
* **Throughput:** 1180.32 Mbps (~1.18 Gbps)
* **Efficiency:** 18.44 Mbps/ALM

## Repository Structure
The repository strictly follows the required EDA environment structure [1, 3]:
* `model/`: Python high-level model and test vector generation scripts.
* `db/`: SystemVerilog RTL design files (`stream_cipher_aes.sv`).
* `tb/`: SystemVerilog testbenches for continuous stream verification.
* `modelsim/`: Scripts and directories (`tv/`, `wave/`) for functional simulation [1].
* `quartus/`: Synthesis constraints (`constr/time_constr.sdc`), virtual pin scripts, and Quartus configurations [2].
* `doc/`: Comprehensive technical project report detailing the architecture and timing protocols [1].

## Prerequisites
To compile and simulate the project, ensure your environment meets the following requirements [4]:
* **Intel Quartus Prime** and **ModelSim** installed.
* Permanent addition of the Quartus (`quartus/bin64`) and Modelsim (`vsim/`) executables to the system `PATH` variable.
* **Python 2.x or 3.x** installed, including `os`, `sys`, `shutil`, and `glob` packages.

## How to Run

### 1. Functional Verification (ModelSim)
1. Open a terminal and navigate to the `modelsim/` directory.
2. Run `python build.py`. This script automatically creates a ModelSim project including all the SystemVerilog files located in the `db/` and `tb/` folders [1].
3. Run the simulation. The test vectors (Key, Plaintext, Ciphertext) will be automatically loaded from the `modelsim/tv/` sub-folder [1].
4. To clean the directory from temporary simulation files, run `python clean.py` [1].

### 2. Logic Synthesis & STA (Quartus Prime)
1. Open a terminal and navigate to the `quartus/` directory.
2. Run `python build.py` and, when prompted by the script, type the top-level name: `stream_cipher_aes` [2].
3. Open the generated `.qpf` project in Quartus Prime.
4. Run **Analysis & Synthesis**.
5. To assign virtual pins, open the menu bar, go to `Tools > Tcl Scripts...`, select `automatic_virtual_pin.tcl` from the `constr/` folder, and click **Run** [2].
6. Run the **Fitter** and the **Timing Analyzer** to extract the implementation metrics.
7. To clean the directory from heavy temporary synthesis files, run `python clean.py` [2].
